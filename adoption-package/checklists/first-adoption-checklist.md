# First Adoption Checklist

Use this checklist before making the public governance baseline a required check in an application repository.

## Wiring

- [ ] `workflows/devsecops-baseline.yml` is copied to `.github/workflows/devsecops-baseline.yml`.
- [ ] The workflow calls `joku-dev/devsecops-governance-framework`.
- [ ] Pull requests and manual runs use `report-only`.
- [ ] Protected `main` runs are the only `block-on-error` path.
- [ ] GitHub Actions artifacts are retained long enough for review.

## Evidence

- [ ] Application artifact path matches the workflow input.
- [ ] SBOM is generated by an agreed tool, not a placeholder.
- [ ] Vulnerability scan is generated by an agreed tool, not a placeholder.
- [ ] Static-analysis results are reviewed by the owning team.
- [ ] `governance/governance-run-input.json` is generated from real repository and pipeline data.

## Repository Controls

- [ ] Default branch is protected.
- [ ] Direct pushes to protected branch are disabled.
- [ ] Pull request review is required before merge.
- [ ] Required checks are stable before enforcement is enabled.
- [ ] Waiver handling is agreed before blocking mode is enabled.

## Architecture Governance

- [ ] `workflows/architecture-governance.yml` is copied when architecture checks are in scope.
- [ ] `.governance/architecture/` evidence files are owned by named roles.
- [ ] Draft evidence is not treated as approved release evidence.
- [ ] Solution baseline name matches the application repository.

## Decision

- [ ] First report-only run reviewed.
- [ ] Findings are triaged.
- [ ] Blocking mode decision is documented.
- [ ] Adoption decision record is completed.
- [ ] Responsible owner is named for recurring evidence maintenance.
