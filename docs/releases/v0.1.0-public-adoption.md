## Purpose

This release marks the first public adoption-ready state of the DevSecOps Governance Framework.

It packages the repository so application teams can consume the public governance baseline from their own repositories without copying governance logic or relying on private source material.

## Included Capabilities

- Public GitHub repository with clean initial history.
- Public GitHub Pages documentation.
- Public Quickstart for application-repository adoption.
- Copyable `adoption-package/` with workflow templates, evidence examples and first-adoption checklist.

- Architecture L1 reusable baseline workflow available at `architecture-baseline-l1-v0.1.0`.
- Validated reference consumption pattern through `joku-dev/ha-CPsWMS`.

## Recommended Consumer Pattern

Application repositories should start with `report-only` governance on pull requests and manual runs.

Blocking behavior should be enabled only after:

- branch protection is configured,
- pull request review is required,
- evidence generation is stable,
- generated findings have been reviewed,
- waiver handling is agreed.

## Known Limitations

- Placeholder SBOM and vulnerability scan examples are for first wiring tests only.
- Architecture evidence placeholders with `status: draft` are not approved release evidence.
- Historical documentation may still describe earlier baseline releases and reference-run history.
- Downstream repositories remain responsible for evidence quality, branch protection and release decisions.
