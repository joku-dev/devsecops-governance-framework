# Documentation Paths

## Purpose

This page organizes the documentation by user journey instead of only by topic.

Use it when you want a guided reading path for a specific role.

## Available Paths

- `beginner-path.md` for first-time readers
- `operator-path.md` for teams running the governance flow operationally
- `auditor-path.md` for reviewers, compliance, and audit stakeholders
- `maintainer-path.md` for repository maintainers and governance engineers

## Quick Selection

Choose this path if you are:

- new to the repository -> `beginner-path.md`
- operating downstream integrations -> `operator-path.md`
- reviewing compliance and evidence -> `auditor-path.md`
- changing baselines, releases, or schemas -> `maintainer-path.md`
