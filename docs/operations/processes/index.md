# Operations Processes

## Purpose

This folder contains governance operation process documents for the DevSecOps
governance repository.

Process documents explain how governance decisions, intake, enforcement,
waivers, timing and result verification are handled operationally. They do not
become source documents by location alone and do not change controls, schemas,
policies or baselines unless a separate governance change explicitly does so.

Use `docs/operations/processes/new-artifact-intake-process.md` before adding a
new artifact type or moving an artifact into a governed repository area.

## Processes

| Process | Purpose |
|---|---|
| `docs/operations/processes/new-artifact-intake-process.md` | First classification process for any new artifact before it enters the governed repository structure. |
| `docs/operations/processes/source-document-intake-process.md` | Report-only process for source-document intake and classification. |
| `docs/operations/processes/source-document-intake-review-operating-model.md` | Human review operating model for source-document intake across ADO and DevSecOps-as-Code. |
| `docs/operations/processes/operational-governance-enforcement-options.md` | Operational modes for report-only, warning and blocking governance enforcement. |
| `docs/operations/processes/waiver-management-standard.md` | Standard waiver format and minimum expectations for controlled exceptions. |
| `docs/operations/processes/application-repo-governance-timing.md` | Timing model between application repositories and the central governance repository. |
| `docs/operations/processes/spot-check-governance-intake.md` | Spot-check strategy for verifying governance intake evidence. |

## Boundary

Practical how-to guides live in `docs/operations/guides/`.

Agent-specific working rules remain outside this folder until the remaining
agent operations material is reviewed separately.
