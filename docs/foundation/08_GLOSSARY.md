# Glossary

## Baseline
A versioned and controlled set of governance requirements, schemas, policies, evidence expectations, and metadata used for reproducible evaluation.

## Capability
A stable function the architecture must provide independently of the product used to implement it.

## Continuous Compliance
Continuous evaluation of engineering evidence against governance requirements.

## Continuous Security
Continuous execution of security controls and generation of security evidence throughout the engineering lifecycle.

## Control
A structured governance requirement with a stable identifier, expected capabilities, evidence, verification method, and exception logic.

## DevSecOps
The operating model that integrates Security into industrialised software engineering.

## Engineering Governance
The organisational and technical capability that establishes justified trust in engineering decisions.

## Engineering Governance Runtime
The operational capability that executes selected governance obligations through controls, evidence, validation, policy evaluation, exceptions, and traceable results.

## Evidence Contract
A technology-independent specification defining required evidence, its producer, subject, structure, provenance, integrity expectations, retention, and supported controls.

## Executable Engineering Governance
The practical execution of selected governance obligations within engineering systems.

## Policy as Code
Executable implementation of objectively verifiable policy or control logic.

## Reference Architecture
A technology-independent description of architectural capabilities, relationships, contracts, and principles.

## Reference Implementation
One concrete implementation used to demonstrate the feasibility of a reference architecture.

## Software Industrialisation
The transformation of software development from project-specific craftsmanship into reproducible engineering systems.

## Trust
Justified confidence in an engineering decision based on authoritative requirements, reliable evidence, controlled evaluation, explicit accountability, and traceable results.

## Waiver
A controlled, authorised, risk-based, and time-bounded exception to a governance requirement.
